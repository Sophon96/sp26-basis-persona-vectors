# Instructor notes — grading rubric, expected outputs, and setup

These notes are for the course staff. They cover: one-time repo setup, expected
outputs per notebook, point weights, and a rubric for the video deliverable.

## One-time repo setup (instructor)

Before students clone:

```bash
# 1. Dependencies
uv sync

# 2. Configure OpenRouter
export OPENROUTER_API_KEY="sk-or-..."
export OPENROUTER_MODEL="anthropic/claude-haiku-4.5"          # judge
export OPENROUTER_ARTIFACT_MODEL="anthropic/claude-sonnet-4.6" # artifact generator

# 3. Generate trait artifacts (~$0.15 total)
uv run python scripts/generate_artifacts.py

# 4. Generate EM-style finetuning datasets (~$0.50 total)
uv run python scripts/generate_em_datasets.py

# 5. On an A100-class GPU: extract persona vectors (~30 min per trait)
uv run python scripts/extract_vector.py --trait evil
uv run python scripts/extract_vector.py --trait sycophancy
uv run python scripts/extract_vector.py --trait hallucination

# 6. Commit the generated artifacts + cached vectors so students start with
#    a working baseline. Do NOT commit `data/cached/judge_scores/` — it's gitignored
#    since the cache can be re-built if needed.
```

After step 6, a student should be able to clone the repo and run notebooks 01
and later without touching OpenRouter for the baseline work.

## Expected outputs per notebook

These are rough sanity checks. Exact values depend on model, seed, and judge
drift — use them as ±10 bands.

| Notebook | Metric | Expected (Qwen2.5-7B, `evil`) |
|----------|--------|-------------------------------|
| 01 | Best layer | 15–20 |
| 01 | Mean trait score at best layer, α=1.5 | ≥ 60 |
| 02 | Pearson r (projection vs trait score) | 0.7–0.85 |
| 03 | Trait score at α=2.0 | ≥ 70 |
| 03 | Coherence at α=2.5 | visibly drops (< 70) |
| 04 | Post-FT trait score (medical_II) | ≥ 40 |
| 04 | Finetuning-shift projection | ≥ 3× baseline |
| 05 | Preventive reduces trait by ≥ 20 pts | yes |
| 05 | Coherence preserved (prev - baseline < 10) | yes |
| 06 | AUROC (projection → flawed label) | ≥ 0.8 |

If a student reports wildly divergent numbers, first check: did they use the
correct `best_layer`? Did the judge API fail silently (returning 0)? Is the
cached rollout set actually populated?

## Point weights (100 pts total)

| Item | Points |
|------|--------|
| Notebook 00 — environment check | 2 |
| Notebook 01 — extraction (correct diff-of-means, reasonable best_layer) | 15 |
| Notebook 02 — monitoring (projection formula, Pearson r reported) | 10 |
| Notebook 03 — steering (α sweep, coherence reported) | 10 |
| Notebook 04 — finetuning shift (shift formula + scatter) | 13 |
| Notebook 05 — preventive steering (both finetunes + comparison) | 15 |
| Notebook 06 — dataset screening (AUROC + qualitative inspection) | 10 |
| Notebook 07 — capstone (completeness + quality of analysis) | 15 |
| Video presentation (see rubric below) | 10 |

## Video rubric (10 pts)

| Dimension | 0 | 1 | 2 |
|-----------|---|---|---|
| Clarity of the question | unclear | stated | well-motivated |
| Method explanation | missing | present | includes a concrete formula |
| Figures | absent/unlabeled | present | annotated + legible |
| Quantitative results | none | reported | with uncertainty/comparison |
| Limitations | unmentioned | acknowledged | thoughtful, specific |

Target length: **5–10 minutes**. Going over doesn't add points.

## Common failure modes

- **Student forgets to record `best_layer`.** Downstream notebooks will error
  cleanly; fix by rerunning notebook 01's TODO 2.
- **Judge returns 0 for everything.** Usually means the OpenRouter key isn't
  exported in the notebook's shell. Verify with a stray `score_trait("test", "evil")`.
- **OOM during finetuning.** Drop `per_device_batch_size` to 1, increase
  `gradient_accumulation_steps`, or use `Qwen2.5-1.5B-Instruct` for development
  and re-run on 7B only for the final submission figure.
- **Capstone scope creep.** Tracks are intentionally bite-sized — 20–30 hours
  total. Push students toward *quantitative comparison* over exploration.

## Notes on reuse of paper materials

- The three trait descriptions in `scripts/generate_artifacts.py` are adapted
  from paper Section 3.1 — short and paraphrased.
- The EM-style datasets generated by `scripts/generate_em_datasets.py` are
  independent regenerations in the spirit of Betley et al. (2025). If you want
  the paper's exact data, point students to
  `github.com/safety-research/persona_vectors`.
